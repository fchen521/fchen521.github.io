##### 造成跨域的原因

```
同源策略是浏览器上为安全性考虑实施的非常重要的安全策略。
简单来说就是协议、端口、和域名有任意一个不同造成的。
```

##### 解决方法

```
解决跨域问题有很多方法，可以通过浏览器，可以同前端代理，也可以后端解决。
对于那些你调用别人的接口，你不能要求别人一定要支持跨域，这个时候只能前端解决
```


###### 浏览器解决Chrome

```
禁止浏览器跨域
1.找到浏览器的安装目录，浏览器快捷键————鼠标右键————属性————目标
2.默认安装位置C:\Program Files (x86)\Google\Chrome\Application
3.在Application下执行打开cmd,或者打开cmd进去到Application目录下
4.执行命令 chrome --disable-web-security
```

##### 被调用方解决

```
Java 后台解决，实现Filter接口（基于Springboot实现的）

@Component
public class CorsFilter implements Filter {
    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
    }
    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {
        HttpServletResponse response = (HttpServletResponse) servletResponse;
        HttpServletRequest request = (HttpServletRequest) servletRequest;
        //动态设置把*号改为request.getHeader("Origin")
        response.setHeader("Access-Control-Allow-Origin","*");
	//根据浏览器上面动态设置
        response.setHeader("Access-Control-Allow-Headers", "Origin, No-Cache, X-Requested-With, If-Modified-Since, Pragma, Last-Modified, Cache-Control, Expires, Content-Type, X-E4M-With, Accept, Authorization, User-Agent");
        //请求的方法
        response.setHeader("Access-Control-Allow-Methods", "HEAD, GET, POST, PUT, DELETE, OPTIONS");
        //OPTIONS域检命令缓存设置过期时间
        response.setHeader("Access-Control-Max-Age", "3600");
        filterChain.doFilter(request, response);
    }
    @Override
    public void destroy() {
    }
}
```

##### 通过nginx方法解决

```
server {
    listen       80;
    server_name  localhost;

	location / {
        proxy_pass   http://ip:port/api/;
        add_header Access-Control-Allow-Methods *;
        add_header Access-Control-Max-Age 3600;
        add_header Access-Control-Allow-Credentials true;
        add_header Access-Control-Allow-Origin $http_origin;
        add_header Access-Control-Allow-Headers $http_access_control_request_headers;

        if ($request_method = OPTIONS ) {
            return 200;
        }
    }
}
```
